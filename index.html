Table of Contents Markdown Generator

Paste your completed documentation here. This application is in its infant stages, and works in a specific way for now. Your documentation must be using Markdown. It also must have its section headers using starting with the ## prefix (it will ignore the top level # prefix to be reserved for the documentation's main header). These will act as the main section links for each section. Any subsequent headers in that section can be set using ###, #### and so on. The application will parse these as sub-sections of the main section.
